Self-hosted instalace

Kompletní Open Source release pod licencí AGPL-3.0. Hostujte si portál na vlastním serveru — zdarma a bez omezení.

Požadavky

  • Linux server s Docker 24+ (2 vCPU, 2 GB RAM pro start)
  • PostgreSQL 16 (spustí se v Docker composu)
  • S3-kompatibilní úložiště (MinIO, Backblaze B2, Cloudflare R2, AWS S3)
  • SMTP relay pro odchozí e-maily
  • Veřejný hostname + TLS (doporučujeme Caddy nebo nginx)

Quickstart (lokální demo)

git clone https://github.com/elh0m3r0/sme-client-portal.git
cd sme-client-portal
cp .env.example .env
docker compose up --build

# v jiném terminálu — připrav demo data
docker compose exec web python -m scripts.seed_dev

# přihlas se na http://localhost:8000
# owner@4mex.cz / demo1234 (staff)
# jan@acme.cz   / demo1234 (kontakt klienta)

Produkční nasazení

V repu je připravený docker-compose.prod.yml pro produkční běh za Nginx proxy s připravenou TLS konfigurací. Stačí doplnit ENV proměnné (SMTP, S3, doména) a docker compose -f docker-compose.prod.yml up -d.

Detaily: docs/SELF_HOST.md, seznam ENV proměnných v docs/ENV.md.

Single-tenant vs. multi-tenant

Self-hostovaná instalace běží ve výchozím stavu jako single-tenant (jeden "dodavatel", více klientských kontaktů). Pokud potřebujete provozovat více oddělených dodavatelů na jedné instanci, zapněte v .env proměnnou FEATURE_PLATFORM=true — aktivuje opt-in SaaS vrstvu s Identity, cross-tenant loginem a admin CRUD pro tenanty.

Podpora

Self-hosters mají přístup k GitHub issues a diskuzím. Pro komerční podporu, audit bezpečnosti nebo custom development využijte kontaktní formulář.
